BioETL is a Python framework for building reproducible bioactivity data pipelines. It connects to major public databases — ChEMBL, PubChem, UniProt, PubMed, CrossRef, OpenAlex, and Semantic Scholar — and delivers clean, analysis-ready datasets through a three-layer medallion architecture: Bronze (raw), Silver (normalized), and Gold (aggregated).

## What BioETL does

BioETL downloads bioactivity data from public APIs, applies schema validation and data quality checks, and writes the results to local Delta Lake tables you can query directly with Polars, Pandas, or any Parquet-compatible tool.

## Get started in 3 steps

<Steps>
  <Step title="Install BioETL">
    Clone the repository and install with `uv` or `pip`.
    ```bash
    git clone https://github.com/SatoryKono/BioactivityDataAcquisition.git
    cd BioactivityDataAcquisition
    make install
    ```
  </Step>
  <Step title="Run your first pipeline">
    Fetch 100 ChEMBL activity records to verify your setup.
    ```bash
    bioetl run --pipeline chembl_activity --limit 100
    ```
  </Step>
  <Step title="Explore your data">
    List all available tables and export results for analysis.
    ```bash
    bioetl export --list
    bioetl export chembl.activity --format csv
    ```
  </Step>
</Steps>

<Note>
  BioETL runs entirely on your local machine. No Docker, Redis, or cloud services are required.
</Note>
